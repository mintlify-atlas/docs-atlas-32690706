## What this template gives you

The sample plugin is a fully working Obsidian plugin that demonstrates the most common extension points in the Plugin API:

- **Ribbon icon** — a clickable icon in the left sidebar that shows a Notice
- **Command palette commands** — three types: simple, editor-scoped, and conditional
- **Settings tab** — a configuration UI that persists data between sessions
- **DOM and workspace event listeners** — registered safely so they clean up on plugin disable
- **Recurring intervals** — managed by the plugin lifecycle so they never leak

## Who this is for

This template is designed for developers building Obsidian community plugins — whether you're making a productivity tool, a custom view, a file manager, or anything in between. You write TypeScript, bundle it with esbuild, and Obsidian loads your `main.js` at startup.

<Note>
  Before building a new plugin, [check if one already exists](https://obsidian.md/plugins) that does what you need. The community has thousands of plugins and you may find a collaborator rather than starting from scratch.
</Note>

## How to get started

<Steps>
  <Step title="Use this template">
    Click **Use this template** on the [GitHub repository](https://github.com/obsidianmd/obsidian-sample-plugin) to create your own copy. This gives you a fresh repo with all the boilerplate in place.
  </Step>
  <Step title="Install dependencies">
    Clone your new repo and run `npm install` to install TypeScript, esbuild, and the Obsidian type definitions.
  </Step>
  <Step title="Start the dev server">
    Run `npm run dev` to compile in watch mode. Edit `src/main.ts` and Obsidian picks up changes after a reload.
  </Step>
  <Step title="Load the plugin">
    Copy `main.js`, `manifest.json`, and `styles.css` into your vault's `.obsidian/plugins/your-plugin-id/` folder, then enable it in **Settings → Community plugins**.
  </Step>
</Steps>

## Key files at a glance

| File | Purpose |
|------|---------|
| `src/main.ts` | Plugin entry point — lifecycle, commands, and event registration |
| `src/settings.ts` | Settings interface, defaults, and the settings tab UI |
| `manifest.json` | Plugin metadata required by Obsidian |
| `styles.css` | Optional CSS bundled with the plugin |
| `esbuild.config.mjs` | Build configuration for bundling TypeScript to `main.js` |
| `versions.json` | Maps plugin versions to minimum Obsidian app versions |
