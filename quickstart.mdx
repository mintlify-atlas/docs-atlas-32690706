---
title: "Quick Start"
description: "Build and install your first Obsidian plugin from the sample template"
---

This guide takes you from the GitHub template to a working plugin loaded in Obsidian.

<Note>
  You need Node.js v16 or later. Run `node --version` to check. Node 18 LTS or newer is recommended.
</Note>

<Steps>
  <Step title="Use the template">
    Open the [obsidian-sample-plugin repository](https://github.com/obsidianmd/obsidian-sample-plugin) on GitHub and click **Use this template → Create a new repository**. This creates your own repo with all the boilerplate in place — you are not forking the original.

    Give your repository a name that matches your intended plugin ID (for example, `my-obsidian-plugin`).
  </Step>

  <Step title="Clone and install dependencies">
    Clone your new repository to your local machine, then install dependencies:

    <CodeGroup>

    ```bash npm
    npm install
    ```

    </CodeGroup>

    This installs TypeScript, esbuild, the Obsidian type definitions, and the ESLint plugin.
  </Step>

  <Step title="Update manifest.json">
    Open `manifest.json` and replace the sample values with your plugin's details:

    ```json manifest.json
    {
      "id": "my-obsidian-plugin",
      "name": "My Obsidian Plugin",
      "version": "1.0.0",
      "minAppVersion": "0.15.0",
      "description": "A brief description of what your plugin does.",
      "author": "Your Name",
      "authorUrl": "https://yourwebsite.com",
      "isDesktopOnly": false
    }
    ```

    Key fields to update:

    | Field | Notes |
    |---|---|
    | `id` | Unique identifier. Must match the folder name in the vault. Never change this after release. |
    | `name` | Human-readable name shown in **Settings → Community plugins**. |
    | `description` | One-sentence description shown in the plugin list. |
    | `author` | Your name or organization. |
    | `minAppVersion` | The minimum Obsidian version your plugin requires. |
    | `isDesktopOnly` | Set to `true` if you use Node or Electron APIs unavailable on mobile. |
  </Step>

  <Step title="Start watch mode">
    Run the development build in watch mode. esbuild compiles `src/main.ts` to `main.js` and recompiles on every save:

    ```bash
    npm run dev
    ```

    Leave this terminal running while you develop. To create a minified production build instead, run `npm run build`.
  </Step>

  <Step title="Install the plugin in your vault">
    Copy the three release artifacts into your vault's plugin folder:

    ```bash
    cp main.js manifest.json styles.css \
      /path/to/your/vault/.obsidian/plugins/my-obsidian-plugin/
    ```

    The folder name must exactly match the `id` field in `manifest.json`. Create the folder if it does not exist.

    <Tip>
      Clone your repository directly into `<Vault>/.obsidian/plugins/<plugin-id>/` instead. When `npm run dev` recompiles `main.js`, the file is already in place — you only need to reload Obsidian, no copying required.
    </Tip>
  </Step>

  <Step title="Enable the plugin">
    In Obsidian, open **Settings → Community plugins**. If prompted, turn off **Restricted mode**. Find your plugin in the list and toggle it on.

    You can also reload the plugin without restarting Obsidian by disabling and re-enabling it from this page.
  </Step>

  <Step title="Test the plugin">
    With the sample plugin enabled, you can verify it is working:

    - **Ribbon icon** — click the dice icon in the left sidebar. A notice reading `This is a notice!` appears.
    - **Command palette** — press `Ctrl+P` (or `Cmd+P` on macOS) and search for "Sample". You will find three registered commands:
      - **Open modal (simple)** — opens a modal from anywhere.
      - **Replace selected content** — replaces the current editor selection with `Sample editor command`. Only visible when a Markdown file is open.
      - **Open modal (complex)** — opens a modal only when a Markdown view is active.
    - **Settings tab** — open **Settings → Community plugins → Sample Plugin → Options** to see the settings tab rendered by `src/settings.ts`.
  </Step>
</Steps>
